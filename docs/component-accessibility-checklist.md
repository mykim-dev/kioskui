# 컴포넌트별 접근성 체크리스트 (키오스크)

이 문서는 키오스크 UI에서 자주 쓰이는 핵심 컴포넌트(버튼/카드/모달/수량조절/결제)에 대해 **제작 시 확인해야 할 체크리스트**를 정리합니다.

---

## 버튼(Button)

- **크기/터치영역**
  - 최소 24×24px 이상, 키오스크는 44×44px 이상 권장
  - 텍스트/아이콘만이 아니라 **버튼 박스 전체**가 클릭/터치 가능해야 함
- **레이블**
  - 텍스트 버튼: 라벨이 **구체적**(예: “확인”보다 “결제 확인”)
  - 아이콘 버튼: `aria-label` 등으로 **의미 있는 이름** 제공(예: “장바구니에서 삭제”)
- **상태**
  - 기본/호버(해당 시)/포커스/활성/비활성 상태가 **명확히 구분**
  - 비활성은 색만으로 구분하지 말고 **명도/테두리/커서/설명** 등으로 중복 표현
- **대비**
  - 버튼 텍스트는 4.5:1(큰 텍스트 3:1) 이상
  - 버튼 경계/아이콘/포커스 링은 배경과 3:1 이상을 목표로
- **키보드**
  - Tab으로 포커스 이동 가능
  - Enter/Space로 활성화 가능

---

## 카드(Card) / 목록 항목(Item)

- **선택 가능 영역**
  - 작은 링크/아이콘만 클릭되게 하지 말고 **카드 전체 클릭**을 우선 고려
- **정보 구조**
  - 제목/가격/설명/태그가 시각적으로 구분되고, 읽는 순서가 자연스러워야 함
  - 스크린리더에서도 의미가 유지되도록 적절한 제목/레이블 구조 제공
- **상태(선택/품절/추천 등)**
  - 색만으로 표현하지 말고 **텍스트 배지(예: “품절”)** 등으로 중복 제공
- **포커스**
  - 카드가 인터랙션 요소라면 포커스 가능해야 하고, 포커스 표시가 명확해야 함

---

## 모달(Modal) / 다이얼로그(Dialog)

- **열림/닫힘**
  - 모달 열릴 때 **포커스를 모달 내부로 이동**
  - 닫힐 때 **원래 트리거 위치로 포커스 복귀**
- **키보드 동작**
  - ESC로 닫기(파괴적 작업 제외)
  - Tab 포커스가 **모달 내부에 갇히도록**(focus trap)
- **제목/설명**
  - 모달 제목은 반드시 존재하고, 스크린리더가 읽을 수 있어야 함(라벨링)
  - 중요한 안내는 텍스트로 제공(색/아이콘만으로 전달하지 않기)
- **닫기 버튼**
  - “X” 아이콘만 두지 말고, 가능하면 **명확한 닫기 버튼**을 함께 제공
  - 아이콘 닫기에도 `aria-label="닫기"` 같은 이름 제공
- **배경 상호작용**
  - 모달이 열린 동안 배경 클릭/스크롤을 제한하여 오작동 방지

---

## 수량조절(Stepper: + / -)

- **터치영역/간격**
  - + / - 버튼은 키오스크에서 특히 오조작이 많으므로 44×44px 이상을 강하게 권장
  - 버튼 사이 간격 확보(붙어있으면 오탭 발생)
- **범위/제한**
  - 최소/최대 값 도달 시 비활성 처리 + **이유 안내(가능하면)**
  - 값이 바뀌었을 때 사용자에게 즉시 피드백 제공(시각 + 필요 시 스크린리더)
- **입력 방식**
  - 숫자 직접 입력이 있으면 키오스크에서는 오입력 위험이 커서
    - 키패드 제공
    - 입력 제한/검증
    - 오류 메시지
    를 함께 설계
- **레이블**
  - “수량”이 무엇의 수량인지 명확히(예: “아메리카노 수량”)
  - 아이콘 버튼은 `aria-label` 제공(예: “수량 증가”, “수량 감소”)

---

## 결제(Payment) / 주문확인(Checkout)

- **핵심 정보 가독성**
  - 총액/할인/결제금액/수량/상품명은 **가장 눈에 띄게**
  - 숫자(금액)는 자리수 변화에도 정렬/줄바꿈이 깨지지 않게
- **오류 예방**
  - 결제/전체 취소/삭제는 확인 단계 제공(오조작 방지)
  - 중복 결제 방지(버튼 연타 방지, 로딩 상태 표시)
- **입력(있다면)**
  - 전화번호/이메일 등 입력은 키오스크에서 부담이 크므로 최소화
  - 입력 필드가 있다면 라벨/예시/오류 메시지를 명확히
- **상태/진행**
  - 결제 처리 중에는 로딩/진행 상태를 명확히 표시
  - 완료/실패 결과를 텍스트로 명확히 안내(색만으로 구분 금지)
- **시간 제한**
  - 결제 단계에서 타임아웃이 있다면 **연장/재시도** 옵션과 충분한 경고 제공

---

## 빠른 셀프 점검(권장 시나리오)

- **키보드만으로**: 첫 화면 → 상품 선택 → 옵션/수량 조절 → 장바구니 → 결제까지 가능?
- **확대 200%**: 주요 화면에서 텍스트/버튼 겹침/잘림/가로스크롤이 생기지 않나?
- **고대비 모드**: 선택/포커스/오류/비활성 상태가 확실히 구분되나?

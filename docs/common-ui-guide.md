# 공통 UI 가이드

이 문서는 무인정보단말기 UI 플랫폼의 공통 UI 가이드 중 **개요 / 화면 레이아웃 가이드 / 사용성 체크리스트**를 개발팀이 바로 적용할 수 있도록 **핵심만 요약**한 문서입니다.
---

## 1) 개요(목적/설명/구성)

### 목적
- **누구에게나 동등한 기회 제공**을 위해 무인정보단말기의 **사용성 및 접근성 보장**이 필요
- 공통 UI 가이드/대분류 UI 가이드를 참고해 **모든 국민이 접근·활용에 어려움이 없도록** 해야 함

### 설명(가이드 제작 근거)
- 국내외 법령 및 표준/가이드 리서치
- 16가지 중분류(64종) 고객 여정 및 UI 분석
- 비장애인·고령자 FGI
- 장애인 FGI
- 접근성 전문가 인터뷰

### 구성
- 무인정보단말기 UI 가이드는 **공통 UI 가이드 + 대분류 UI 가이드(유통/주문/발권/안내 및 기타)**로 구성
- 효과적인 사용 경험 제공을 위한 **인터페이스 설계 방법**을 제시

---

## 2) 화면 레이아웃 가이드(핵심 요약)

### 큰 원칙
- **공통된 레이아웃**은 사용자가 과업에서의 현재 위치를 파악하고 과업 완료까지 도달할 수 있게, **빠르고 체계적인 흐름**을 제공해야 함
- 단말기 “개별 화면의 일관성”뿐 아니라 “다른 무인정보단말기 간의 일관된 사용자 경험”도 고려해 설계

### 레이아웃 일관성
- 화면 요소는 **위→아래, 좌→우**의 논리적 순서를 지키는 것이 정보 파악에 유리
- 시각장애 사용자는 **수직 계열화(위→아래)** 개념이 익숙하므로 정보 구조를 위→아래로 제공하는 것을 권장
- 모든 설계 요소는 **단순하고 일관성 있게** 배치

### 한 화면 = 한 과업
- 사용자 부담을 줄이기 위해 **한 화면에 하나의 과업**을 제공
- 과업 도중 **추천/광고/새 과업 제안**은 흐름을 끊을 수 있으므로 지양

### 권장 화면 영역(상단/단계/정보·컨트롤/하단)
- **상단 공통 영역**
  - 좌측: 처음 화면(홈)으로 가기 컨트롤
  - 우측: 직원 호출 컨트롤
  - 선택: BI 로고, 언어 변경 컨트롤
  - 상단은 **간결하게**, 불필요한 꾸밈/과다 정보 지양
  - 상단 공통 영역은 **항상 동일 위치·동일 형태**로 제공
- **단계 표시 영역**
  - 사용자의 **현재 위치/진행 단계**를 알려주는 영역
- **정보 영역 및 컨트롤 영역**
  - 과업 수행에 필요한 정보 + 과업 완료를 위한 주요 컨트롤
  - 제목/안내 문구를 **정확하고 잘 보이게** 표시
  - 정보 영역과 컨트롤 영역을 **명확히 구분**
  - 불필요한 꾸밈/과다 정보 지양
- **하단 공통 영역**
  - 좌측: 이전으로 가기 컨트롤
  - 우측: 장애인/고령자 접근성 기능 컨트롤
  - 과업 중 필요한 기능은 **쉽게 접근** 가능해야 함

### 화면 크기 권장
- 접근성(글자 크기/컨트롤 크기·간격) 관점에서 **27인치 이상 화면**을 권장

## 3) 사용성 체크리스트(실무 적용용)

아래 항목은 페이지의 체크리스트를 **개발 관점에서 바로 점검**할 수 있게 재구성한 것입니다. (원문 항목 기반)

### 레이아웃/진행
- [ ] **정보 영역·컨트롤 영역**을 사용자의 **중심 시야각**에 배치
- [ ] 과업 진행 **단계 표시** 제공(현재 위치/남은 단계 예측 가능)

### 입력/조작(터치)
- [ ] **멀티터치(두 손가락 이상 동작)**을 요구하지 않기(연속 단순 터치로 대체)
- [ ] 화면/표면의 모든 글자 **높이 12mm 이상**
  - 확대 시 글자가 가늘어지거나 깨지지 않게
  - 산세리프(획 삐침이 없는) 서체 권장
- [ ] 컨트롤 크기/간격
  - [ ] **면적 150mm² 이상**
  - [ ] **한 변 12mm 이상**
  - [ ] 인접 컨트롤 간 **간격 2.5mm 이상**

### 대비/가독성/시각적 구분
- [ ] 의미 있는 글자/아이콘/컨트롤은 배경과 **명도 대비 명확**
  - [ ] 기본 **4.5:1 이상**, 큰 글자(12mm 초과) **3:1까지 허용**
  - [ ] **녹색 바탕+빨간 글자**, **파란 바탕+노란 글자** 조합은 사용하지 않기
- [ ] 인접 콘텐츠는 **시각적으로 명확히 구분**
- [ ] 불필요한 이미지 삽입 지양(가독성 방해/중요정보 파악 지연)
- [ ] 아이콘/픽토그램은 **표준적인 모양** + **텍스트 병기**
- [ ] 용어는 **일상적이고 자연스러운 표현** 사용(전문용어/약어/외래어는 대체 또는 쉬운 설명 추가)
- [ ] 색은 강조용으로 사용하고 **패턴/레이블 등 중복 단서** 제공(색만으로 구분 금지)

### 탐색/페이지 구성
- [ ] 카테고리가 한 줄을 넘으면 좌우 스와이프보다 **2줄 카테고리** 적용
- [ ] 긴 화면은 **페이징 우선**, 필요 시 스크롤 병행(스크롤은 시각장애 사용자에게 어지러움 유발 가능)
- [ ] 페이지가 여러 장이면 **명확한 페이지 표시**
- [ ] 언제든 **이전/처음(홈)**으로 이동 가능한 컨트롤 제공(취소/초기화 가능)
- [ ] **직원 호출** 컨트롤 제공(대체로 연락처를 숫자+촉각(점자)로 표시)

### 자동 변화/시간 제한/점멸
- [ ] 사용자 조작 없이 자동으로 변경되는 콘텐츠 지양(부득이하면 제어 수단: 앞/뒤/건너뛰기/일시정지 제공)
- [ ] 사용자 선택 응답에 **시간 제한(timeout) 미부여**
  - 부득이하면 남은 시간 표시 + 연장 컨트롤 제공
- [ ] 깜빡임/번쩍임 객체 미사용
  - 사용한다면 **초당 3~50회 주기 점멸 금지**

### 접근성 기능(장애인/고령자)
- [ ] 접근성 기능 제공(예: 낮은 화면 모드, 고대비, 확대, 수어 안내, 음성 안내 등)
- [ ] **고대비 화면** 제공 + 사용자가 **쉽게 활성화** 가능
  - 고대비 모드에도 일반 모드 기능이 **동등하게 포함**
- [ ] 의미 있는 콘텐츠는 **대체 콘텐츠** 제공(예: 점자/음성, 수어/자막 등)
  - 이미지/영상에는 정확한 대체텍스트 제공
- [ ] 의미 있는 모든 시각 정보는 **음성 안내** 제공
  - 최대 음량 **65dBA 보장**
  - 음량 조절 범위 **12dB~18dB**
  - 다시 듣기 기능 제공
  - 사용자 선호에 따라 음성 안내 활성화/차단 가능
- [ ] 점자 표시 제공(카드 투입구/티켓 배출함 등 디바이스 용도 식별)
- [ ] 수어 안내 제공(단어 나열이 아니라 화면별 상황/맥락 중심 안내)

### 설치/물리 환경(높이/도달)
- [ ] 컨트롤 영역 위치: 지면에서 **400mm~1,220mm**
  - “낮은 화면 모드” 등으로 휠체어 사용자 고려(중요·고빈도 기능은 하단 배치)
- [ ] 화면 내 시각 정보 위치: 바닥으로부터 **1,220mm 초과 금지**
  - 예외: 로고/화면 제목 등 과업에 중요하지 않은 정보

### 피드백/오류
- [ ] 중요한 입력 완료 시 **피드백(색/명도 변화 + 소리/진동 등)** 제공
  - 확인 버튼이 필요한 흐름이면 확인 버튼 상태 변화 + 음향/진동으로 완료 알림
- [ ] 입력 오류 정정 방법 제공 + 오류 원인에 대한 **정확한 시각/비시각 안내**
  - 경고음은 점멸/불빛 등 시각 효과와 함께 제공

### 개인정보/인증/보호
- [ ] 개인정보(개인식별번호 등)는 시스템이 임의로 정하지 않고 **사용자가 직접 설정**
- [ ] 생체 인식 기반 식별/제어 시 **다양한 인식 방법** 제공(하나 이상) + 사용 방법 가이드 제공
- [ ] 입력/출력되는 개인정보는 보호
  - 화면에 평문 노출 금지(암호 형식 표시)
  - 음성정보는 사용자가 동의한 장치로만 제공(외부로 읽어주지 않기)

### 다국어
- [ ] 지원되는 모든 외국어는 한국어와 **동등한 수준의 접근성 서비스** 제공

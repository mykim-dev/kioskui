WCAG 2.x 접근성 가이드예요.

🎯 기본 원칙 (WCAG AA 기준)
텍스트 크기	정의	최소 대비 비율
일반 텍스트	본문 대부분의 글자	4.5 : 1
큰 텍스트	24px 이상 또는 18.66px 이상 + Bold	3 : 1
UI 요소/아이콘	버튼 테두리, 아이콘 등	3 : 1

✔ 키오스크는 고령자 사용을 고려해 가능하면 모든 텍스트를 4.5:1 이상으로 잡는 것이 안전합니다.

🔤 “큰 텍스트”의 기준이 헷갈릴 때

WCAG에서 말하는 Large text는:

일반 굵기 → 24px 이상 (약 18pt)

Bold(700) → 18.66px 이상 (약 14pt Bold)

즉,

예시	큰 텍스트 여부	적용 대비
20px Regular	❌ 일반 텍스트	4.5:1
24px Regular	✅ 큰 텍스트	3:1
18px Bold	❌ 일반	4.5:1
19px Bold	✅ 큰 텍스트	3:1
📌 AAA까지 고려하면
텍스트 크기	AA	AAA
일반 텍스트	4.5:1	7:1
큰 텍스트	3:1	4.5:1

👉 실무/공공 키오스크: AA 이상이면 대부분 충족

🖥️ 키오스크 실무 적용 팁

거리 & 환경

사용자는 보통 50~100cm 거리에서 봄 → 실질적으로 “작은 글자”는 더 작게 인식됨
→ 큰 텍스트라도 4.5:1 유지 권장

야외/저휘도 모드

밝기 낮아지면 체감 대비 감소
→ 다크모드에서도 명도 차 충분히 확보

얇은 폰트 주의

Light/Thin은 같은 px라도 더 흐리게 보임
→ Regular 이상 + 높은 대비 권장

아이콘/버튼

텍스트 없으면 형태 대비 3:1 이상 필수

✅ 한 줄로 정리하면

작은 글자(본문) → 4.5:1 이상
큰 글자(24px↑ 또는 Bold 19px↑) → 3:1 이상 가능
👉 하지만 키오스크는 가급적 전부 4.5:1 이상이 가장 안전

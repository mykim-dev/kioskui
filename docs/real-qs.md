## 🖥️ 무인정보단말기 UI 퍼블리싱 핵심 체크리스트

### 1. 📐 화면/디바이스 환경

* **해상도 & 비율**: 고정 해상도인지, 다기종 대응인지 명확히
* **가로/세로 방향**: 회전 여부, 고정 여부
* **베젤/실사용 영역**: 디자인 대비 실제 터치 영역 차이 확인
* **밝기/야외 사용**: 다크모드, 저휘도 모드 필요 여부

👉 *리스크*: 디자인 그대로 구현했는데 실제 기기에서 잘림/왜곡 발생

---

### 2. 👆 터치 인터랙션 기준

* **최소 터치 영역**: 버튼/입력 최소 48px 이상
* **Hover 개념 없음** → 모든 UX는 *탭 기준*
* **멀티터치 지원 여부**
* **터치 피드백**: 시각적 반응 필수

👉 *리스크*: PC 기준 UI → 현장에서는 오작동/오터치 다발

---

### 3. ♿ 접근성 & 법적 요구

* **고대비/가독성**: WCAG AA 수준 대비
* **색상만으로 정보 전달 금지**
* **큰 글자/간단한 문장**
* **음성/안내 연계 여부**

👉 *리스크*: 납품 후 접근성 미준수로 재작업/클레임

---

### 4. 🔤 폰트 & 리소스

* **폰트 제공 주체**: 고객? 시스템? 라이선스?
* **웹폰트 vs 로컬폰트**
* **이미지/아이콘 포맷 & 해상도**
* **저사양 기기에서의 용량/로딩 고려**

👉 *리스크*: 라이선스 문제, 성능 저하, 현장 폰트 깨짐

---

### 5. ⚙️ 실행 환경 & 기술 스택

* **OS**: Android / Windows / Linux / WebView?
* **브라우저/엔진 버전**
* **네트워크**: 오프라인 동작 필요 여부
* **ES/CSS 지원 범위**

👉 *리스크*: 최신 CSS/JS 사용 → 기기에서 미지원

### 6. 🚀 성능 & 안정성

# 키오스크 유형 가이드 (유통형 / 주문형 / 발권형 / 안내 및 기타)

이 문서는 무인정보단말기 UI 플랫폼의 대분류 관점에서 **키오스크 유형을 4가지로 구분**하고,
각 유형별로 **단말기(설치 맥락) 예시, 대표 과업 흐름(steps), 화면 구성 예시(샘플 화면)**를 정리한 문서입니다.

> 목적: “같은 키오스크”라도 **유형별로 사용자가 기대하는 흐름/정보/결제/발권/안내 패턴이 다르기 때문에**  
> 설계·구현 단계에서 공통 레이아웃을 유지하되, 유형 특성에 맞게 step/정보 구조를 잡을 수 있도록 돕습니다.

---

## 1) 유통형(상품코드 스캔 기반 구매)

### 단말기(종류) 예시
- 유통형 무인정보단말기는 **상품코드(바코드 등)를 사용자가 직접 스캔해서 제품을 구매**하는 유형이다. (출처: 무인정보단말기 UI 플랫폼)
- 대표 단말 예시:
  - **무인주유기**
  - **무인주차정산기**
  - **무인결제기**
  - **무인도서대여반납기**
  - **무인발권기(사진)**
  - **무인상품판매기**
  - **무인주유기(충전)**

### 대표 사용자 목표
- 상품/서비스를 선택(또는 스캔)하고 **내역을 확인한 뒤 결제**까지 완료하기
- 실수(중복 스캔/잘못된 항목/수량 오류)를 줄이기 위해 **중간 내역 확인 단계**가 중요

### step 예시(대표 흐름)
유통형 공통 프로세스는 아래 5단계를 기본으로 볼 수 있습니다.  
(출처: 무인정보단말기 UI 플랫폼 — 공통 UI 가이드/유통형)

1. **시작안내**
2. **기본선택**
3. **내역확인**
4. **결제하기**
5. **완료**

## 2) 주문형(음식/카페/푸드코트/패스트푸드)

### 단말기(종류) 예시
- 주문형 무인정보단말기는 **직원의 도움이 필요한 음식물 등의 주문 및 구매**하는 유형이다. (출처: 무인정보단말기 UI 플랫폼)
- 대표 단말 예시:
  - **무인주문기**

### 대표 사용자 목표
- 메뉴를 고르고(옵션 포함) **주문 내용을 정확히 확인**한 뒤 결제하고, **완료(주문번호/안내)**까지 도달하기

### step 예시(대표 흐름)
주문형 공통 프로세스는 아래 6단계를 기본으로 볼 수 있습니다.  
(출처: 무인정보단말기 UI 플랫폼 — 공통 UI 가이드/주문형)

1. **시작안내**
2. **기본선택**
3. **옵션선택**
4. **내역확인**
5. **결제하기**
6. **완료**

## 3) 발권형(교통/공공/공연/대기표/주차정산)

### 단말기(종류) 예시
- 발권형 무인정보단말기는 **요청에 대해 정보처리 후 인쇄물을 제공**하는 유형이다. (출처: 무인정보단말기 UI 플랫폼)
- 발권형 무인정보단말기에는 아래 예시가 있다:
  - **무인민원발급기**
  - **무인발권기**
  - **무인발매기**
  - **무인증명발매기**
  - **셀프체크인**
  - **무인처방전발매기**
  - **금융자동화기기**
  - **무인결제기(중고폰판매)**
  - **무인발권기(기차)**
  - **무인발권기(버스)**
  - **무인발매기(박물관)**
  - **무인발매기(스포츠)**
  - **셀프체크인(숙소)**

### 대표 사용자 목표
- 필요한 정보를 입력/처리하고(필요 시 본인확인 포함), **내역 확인 후 결제 및 발권(인쇄물 수령)**까지 완료하기

### step 예시(대표 흐름)
발권형 공통 프로세스는 아래 8단계를 기본으로 볼 수 있습니다.  
(출처: 무인정보단말기 UI 플랫폼 — 공통 UI 가이드/발권형)

1. **시작안내**
2. **기본선택**
3. **본인확인**
4. **옵션선택**
5. **내역확인**
6. **결제하기**
7. **발권**
8. **완료**

## 4) 안내 및 기타(정보 제공 중심 + 기타 단말)

### 단말기(종류) 예시
- 안내형 무인정보단말기는 **사용자 조작에 따라 정보를 제공해주는** 유형이다.  
  안내형은 목적 달성을 위한 절차적 진행보다 **정보 습득을 위한 다양한 경로**가 특징이다. (출처: 무인정보단말기 UI 플랫폼)
- 안내형 무인정보단말기 예시:
  - **종합정보시스템**
  - **위치정보시스템**
  - **무인사용자인증기(예약)**
  - **종합정보시스템(관광)**
  - **종합정보시스템(길안내)**
- 기타 유형 예시:
  - **무인사용자인증기**
  - **사물함**
  - **택배함** 등

### 대표 사용자 목표
- 빠르게 정보를 탐색해(경로/층/시설/관광 등) **세부 안내**까지 도달하고, 필요하면 추가 기능(출력/QR 등)까지 수행하기

### step 예시(대표 흐름)
안내형은 아래 2단계로 구성됩니다.  
(출처: 무인정보단말기 UI 플랫폼 — 공통 UI 가이드/안내 및 기타)

1. **안내**
2. **세부안내**

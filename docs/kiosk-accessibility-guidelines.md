# 키오스크 UI 접근성 제작 가이드 (요약)

이 문서는 `README.md`의 참고 자료(무인정보단말기 UI 플랫폼, “웹접근성을고려한콘텐츠제작기법 2.0”, “KWCAG 2.2”)를 바탕으로 **키오스크 UI 제작 시 반드시 인지해야 할 항목**을 실무 체크 형태로 정리한 요약본입니다.

---

## 터치 타깃(컴포넌트) 크기 / 간격

- **최소 터치 타깃**
  - **24×24px 이상**(최소 기준으로 많이 사용)
  - 키오스크는 오조작/거리 사용을 고려해 **44×44px 이상 권장**
- **아이콘만 작게 클릭**시키지 말 것
  - 아이콘 버튼이라도 **버튼 박스 자체**를 크게(패딩 포함) 만들어 클릭 영역을 확보
  - 리스트/카드 선택은 가능하면 **행/카드 전체 클릭**으로 설계
- **타깃 간 간격**
  - 인접 요소가 붙어 있으면 오조작이 증가하므로 **충분한 여백(간격)**을 확보
  - “+ / -” 같이 반복 조작 요소는 특히 간격과 크기를 더 크게

---

## 텍스트 크기 / 가독성

- **기본 폰트 크기**
  - 일반적으로 **16px 이상**을 기본으로 권장
  - 키오스크는 환경(거리/조명)을 고려해 **더 큰 기본 크기**를 기본값으로 검토
- **정보 우선순위에 따른 크기 체계**
  - 가격/총액/결제 금액, 다음 단계 CTA 등 핵심 정보는 **명확히 더 크게**
- **행간/자간**
  - 장문/설명 텍스트는 **행간을 넉넉히** 하여 읽기 쉽게 유지

---

## 색 대비(대비율) / 상태 표현

- **텍스트 대비율**
  - 일반 텍스트: **4.5:1 이상**
  - 큰 텍스트(큰 글자/굵은 글자 등): **3:1 이상**
- **비텍스트(UI 요소) 대비**
  - 버튼 윤곽/아이콘/포커스 링/입력 테두리 등은 배경과 **3:1 이상**을 목표로
- **색만으로 상태를 전달하지 말 것**
  - 선택/오류/성공/비활성은 **색 + 아이콘 + 텍스트(또는 형태/패턴)**로 중복 제공
- **고대비 모드 제공**
  - 고대비 모드(혹은 테마 전환)는 시각 접근성에 도움이 되므로 **쉽게 켜고 끌 수 있게**
  - 모드 전환 시에도 브랜드 컬러보다 **가독/식별(대비) 우선**

---

## 확대·축소(Zoom) / 글자 확대 대응

- **브라우저 확대(예: 200%)에도 사용 가능**
  - 텍스트/버튼이 겹치거나 잘리지 않도록 레이아웃이 **재배치**되게 설계
  - 가로 스크롤 강제는 지양(불가피하면 핵심 흐름은 유지)
- **텍스트만 확대(글자 크기 조절) 시에도 UI가 함께 따라가야 함**
  - 글자만 커져서 버튼/카드 안에서 잘리면 실패
- **키오스크 환경 고려**
  - 브라우저 제스처/단축키 사용이 제한될 수 있으니, 필요하면
    - “글자 크게/작게”
    - “화면 확대/기본”
    같은 UI 제공을 검토

---

## 키보드/포커스/스크린리더(기본 접근성)

- **키보드로 전 기능 조작 가능**
  - Tab/Shift+Tab 이동, Enter/Space 활성화, ESC 닫기(모달) 등
- **포커스 표시가 명확해야 함**
  - 현재 위치가 한눈에 보이는 **두드러진 포커스 링**(색 대비 포함)
- **이름(레이블) 제공**
  - 아이콘 버튼/수량조절/삭제 등은 **의미 있는 접근성 이름(aria-label/aria-labelledby)** 제공
- **상태 변화 알림**
  - “장바구니 담김/총액 변경/오류”는 시각적 표시뿐 아니라, 필요 시 **스크린리더 알림(live region 등)**을 검토

---

## 시간 제한/애니메이션/환경

- **세션 타임아웃/자동 초기화가 있다면**
  - 사용자에게 **연장/취소 선택**을 제공하고, 충분히 큰 경고를 제공
- **과도한 움직임/점멸 금지**
  - 빠른 점멸은 발작 유발 위험이 있으므로 피함
  - 애니메이션은 인지 부담을 줄이도록 절제하고, 가능하면 감소 옵션을 고려

---

## 오류 예방/인지 부담 감소(주문·결제 핵심)

- **중요 행동은 확인 단계 제공**
  - 결제/전체취소/삭제 등은 오조작 방지를 위해 확인 단계 및 되돌리기(가능하면)를 제공
- **오류 메시지는 해결 방법까지**
  - “무엇이 문제인지 / 어떻게 해결하는지”를 **텍스트로 구체적**으로 안내
- **일관성 유지**
  - 버튼 위치(이전/다음/결제), 용어, 흐름이 화면마다 바뀌지 않게 유지
